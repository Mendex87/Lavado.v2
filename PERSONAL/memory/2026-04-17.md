- Decisión clave de arquitectura para la app/planta: diseñar todo con tolerancia a fallas de balanzas y señales. Aunque el ideal es automatización total vía PLC, el sistema debe permitir seguir operando cuando una balanza quede fuera de línea o falle, incluyendo ingreso manual de valores procesados por usuario. También se pidió que la integración futura de nuevas balanzas o puntos de medición sea fácil y escalable, sin depender de un esquema rígido atado solo a las balanzas actuales.
- Se definió una arquitectura flexible para integración PLC-app: measurement points configurables por línea/proceso, con source_mode `plc`, `manual` o `mixed`, para permitir crecimiento futuro y operación degradada cuando fallen balanzas o señales.
- Regla funcional importante del proyecto: no asumir automatización perfecta; la app debe seguir operando si una balanza falla, permitiendo carga manual y fallback operativo.
- Se decidió implementar la conexión al Siemens S7-1200 con un proceso Python separado dentro del mismo repo (`backend/plc_poller`), evitando por ahora un dispositivo o servicio intermedio como Node-RED.
- Quedó implementada en backend la base de `measurement_points` y `measurement_readings`, con endpoints `GET /api/v1/measurements/points` y `POST /api/v1/measurements/ingest`.
- Se armó la base del poller Python para S7 usando `python-snap7`, con mapping configurable y publicación al backend.
- Se validó una primera integración real en la PC del usuario, dentro de la misma red del PLC: conexión exitosa al S7-1200 en `192.168.10.77`, rack 0 slot 1.
- Se confirmó lectura real de DBs del PLC con acceso no optimizado: `DB13` para parciales y `DB2` para totalizadores.
- La primera prueba real quedó mapeada como `l1_input_main`, leyendo `DB13 byte 0` (partial) y `DB2 byte 0` (totalizer).
- Se verificó el flujo completo `PLC -> poller Python -> backend -> base de datos`, con una ingesta exitosa (`readings_created: 1`).
- Se detectó que la UI actual (`app-preview`) es una preview estática servida con Python (`index.html`, `app.js`, `styles.css`) y todavía no consume ni muestra las nuevas lecturas reales del PLC; próximo paso lógico es exponer lecturas recientes y conectarlas en `app.js`.
